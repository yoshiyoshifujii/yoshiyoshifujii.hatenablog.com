さっそく、無線LANで接続できるようにしたので、メモ。

こちら、参考にさせていただきました。ありがとうございました。
[http://openrtm.org/openrtm/ja/node/266#toc2:title]

[asin:B003NSAMW2:detail]

以下の手順だよ。

1. /etc/network/interfaces の編集
2. /etc/wpa_supplicant/wpa_supplicant.conf の編集
3. ホスト名で接続

====

** 1. /etc/network/interfaces の編集

>|zsh|
$ sudo vi /etc/network/interfaces
||<

で開いて、

>|conf|
iface wlan0 inet manual
↓
iface wlan0 inet dhcp 
||<

と、

>|conf|
wpa-roam /etc/wpa_supplicant/wpa_supplicant.conf
↓
wpa-conf /etc/wpa_supplicant/wpa_supplicant.conf
||<

って感じ。

** 2. /etc/wpa_supplicant/wpa_supplicant.conf の編集

以下のコマンドを実行する。

ESSIDと、passのところは、御自身の環境に合わせてね。

>|zsh|
$ sudo bash
# cd /etc/wpa_supplicant
# wpa_passphrase ESSID pass >> wpa_supplicant.conf
||<

うちのNWは、これだけで接続できたけど、環境によっては、もそっと設定ファイル調整してやる必要があるらしい。

** 3. ホスト名で接続

ほいで、いっつもDHCPからIPアドレスさがすのとか、固定にしちゃって他のとこで使えないとかを避けるために、ホスト名で接続できるようにする。

>|zsh|
$ sudo vi /etc/hostname 
$ sudo vi /etc/hosts
||<

で、ホスト名を好きなんつけてやる。

ほいでさらに、

>|zsh|
$ sudo apt-get install avahi-daemon
||<

で、avahi-daemonをインストールしたる。

>|zsh|
$ sudo reboot
||<

でマシン再起動したったら、

>|zsh|
$ ssh pi@hostname.local
||<

って感じで接続できるようになったはる。

hostnameのとこは、もちろん、君が付けたやつだよ。

以上。
